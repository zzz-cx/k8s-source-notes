# 字节跳动技术面试 — 综合准备指南

> 适用：飞书稳定性 / SRE / 后端基础设施方向（2027 届实习）  
> 官方要求：**算法 + 数据结构 + 系统设计 + 岗位领域知识**

---

## 一、官方要求解读

字节技术面试考察三块，准备时需 **并行推进**，不可只背八股：

| 模块 | 考察点 | 你的准备动作 |
|------|--------|-------------|
| **算法 & 数据结构** | 白板写 **可执行代码**，分析时空复杂度，考虑边界与副作用 | LeetCode 按专题刷 + 白板模拟 |
| **系统设计** | 需求 → 架构，比较方案优劣，在约束下选型，可扩展性 | 按框架练 + 读 High Scalability |
| **领域知识** | 岗位相关技术（稳定性：网络/LB/微服务/监控/K8s 等） | 本仓库问答手册 + K8s 笔记 |
| **运维脚本** | Linux / Bash / Python 自动化与排障 | [ops-scripting-interview-qa.md](./ops-scripting-interview-qa.md) |

### 面试官期望的行为

1. **即时沟通思考过程**，不要闷头写 20 分钟  
2. 主动 **澄清需求**（规模、QPS、延迟、一致性要求）  
3. 尝试 **多种解法**，对比优劣后再定稿  
4. 代码必须是 **真实可运行代码**，不是伪代码  
5. 写完主动分析 **时间/空间复杂度** 和 **潜在问题**

## 二、准备清单（按模块）

### ✅ 模块 A：软件工程 & 系统设计

#### 核心能力（官方原文对应）

| 能力 | 含义 | 如何练 |
|------|------|--------|
| 需求 → 设计 | 从模糊需求提炼功能、非功能需求 | 每题先写：用户、场景、约束 |
| 衡量设计好坏 | 可用性、延迟、成本、复杂度、运维 | 用表格对比 2～3 套方案 |
| 约束下选型 | 人/时间/资源有限时选 **够用且可演进** 的方案 | MVP + 迭代路径 |
| 可扩展性 | 水平扩展、解耦、无状态、分片 | 设计里单独一节 Scalability |

#### 系统设计答题框架（建议背熟）

```
1. 澄清需求（5 min）
   - 功能：核心用例
   - 规模：DAU、QPS、数据量、读写比
   - 非功能：延迟、可用性、一致性

2. 高层架构（10 min）
   - 客户端 → LB → 网关 → 服务 → 存储
   - 画框图，标数据流

3. 核心组件深入（15 min）
   - API 设计、DB  schema、缓存策略
   - 关键路径时序

4. 扩展与优化（10 min）
   - 分库分表、缓存、MQ、CDN
   - 瓶颈在哪、如何扩

5. 可靠性（稳定性岗加分）
   - 监控 SLI/SLO、降级、限流、多 AZ
   - 故障场景与预案

6. 总结权衡
   - 为什么选 A 不选 B
```

#### 稳定性岗常见系统设计题

| 题目 | 练习重点 |
|------|----------|
| 设计 IM 消息系统 | 长连接、顺序、已读、离线推送 |
| 设计短 URL 服务 | 哈希、冲突、301/302、QPS |
| 设计限流器 | 令牌桶/漏桶、分布式限流 |
| 设计监控告警系统 | Metrics 采集、存储、告警规则 |
| 设计分布式任务调度 | 队列、幂等、重试、分片 |
| 设计飞书文档协同 | OT/CRDT 概念、冲突、实时性 |

#### 设计对比示例（面试话术模板）

> 「方案 A 用 MySQL 单库，实现简单，适合初期 QPS < 1k；方案 B 读写分离 + Redis 缓存，读扩展更好，但有一致性窗口。当前阶段我选 A + 预留分库接口，因为团队小、上线紧，且读压力可通过缓存层后续加。」

### ✅ 模块 B：算法 & 数据结构

#### 要求对照

| 官方要求 | 具体做法 |
|----------|----------|
| 白板清晰简洁 bug free | 纸笔 / 白板每天 1 题，限时 25 min |
| 时空复杂度 | 每题写完必 verbally 分析 |
| 多种解法 | 每题至少想：暴力 → 优化 → 最优 |
| 可执行代码 | 用 **Python 或 Go** 在 LeetCode 提交通过 |
| 沟通思路 | 录音自问自答，或找同学 mock |

#### 推荐专题顺序（LeetCode）

| 阶段 | 专题 | 典型题 | 目标 |
|------|------|--------|------|
| 1 | 数组 / 哈希 | 两数之和、三数之和、最长无重复子串 | 双指针、哈希 O(1) 查 |
| 2 | 链表 | 反转链表、合并 K 个链表、环检测 | 虚拟头节点、快慢指针 |
| 3 | 栈 / 队列 | 有效括号、每日温度、滑动窗口最大值 | 单调栈 |
| 4 | 二叉树 | 层序遍历、LCA、验证 BST | DFS/BFS 模板 |
| 5 | 二分 | 搜索旋转数组、找峰值 | 边界 mid 写法 |
| 6 | 动态规划 | 爬楼梯、背包、LIS | 状态定义 + 转移 |
| 7 | 图 | BFS 最短路径、课程表（拓扑） | 邻接表、visited |
| 8 | 堆 / TopK | 第 K 大、合并数据流 | 小顶堆/大顶堆 |

#### 字节高频算法模式

- 双指针、滑动窗口  
- BFS/DFS（树、图）  
- 二分答案  
- 前缀和 / 差分  
- 单调栈  
- DP（一维为主，实习难度）  
- 堆（TopK、多路归并）  

#### 白板代码规范

```python
# 1. 先写函数签名和注释说明思路
# 2. 处理边界：空输入、单元素、负数
# 3. 主逻辑
# 4.  verbally: 时间 O(n)，空间 O(1)，无原地修改副作用

def solution(nums: list[int]) -> int:
    if not nums:
        return 0
    # ...
    return ans
```

#### 复杂度速查

| 结构/操作 | 平均 |
|-----------|------|
| 哈希查/插 | O(1) |
| 二叉搜索树 | O(log n) |
| 堆 push/pop | O(log n) |
| 排序 | O(n log n) |
| BFS/DFS | O(V+E) |

#### 副作用 checklist

- 是否修改了输入数组？  
- 递归深度是否栈溢出？  
- 整数溢出？（用 `//` 或 long）  
- 空指针 / 越界？

## 六、面试当天提醒

1. **想大声说出来**，把面试官当结对编程伙伴  
2. 卡壳时：**「我先用暴力 O(n²)，再尝试用哈希优化到 O(n)」**  
3. 设计题：**先广后深**，不要一上来钻 DB 索引  
4. 稳定性岗：**主动提可用性、降级、监控**，这是差异化  
5. 结束前准备 1～2 个 **有质量的问题** 问面试官
